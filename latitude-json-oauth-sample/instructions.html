Instructions for the Google Latitude API JSON OAuth Sample

Browse Online

  • Browse Source, or main file LatitudeSample.java

Checkout Instructions

Prerequisites: install Java 6, Mercurial and Maven. You may need to set your JAVA_HOME.

Important: after checking out the project, and before compiling and running it, you need to edit src/com/google/api/client/sample/latitude/model/ClientCredentials.java to enter a domain and client shared secret for a registered domain that you own. Otherwise, you will get a 400 Bad Request exception with the message Consumer is not registered: enter_domain.

cd [someDirectory]
hg clone https://samples.google-api-java-client.googlecode.com/hg/ google-api-java-client-samples
cd google-api-java-client-samples/latitude-json-oauth-sample
[editor] src/com/google/api/client/sample/latitude/ClientCredentials.java
mvn compile
mvn -q exec:java

Setup Project in Eclipse 3.5/3.6

Prerequisites: install Eclipse, the Mercurial plugin, and the Maven plugin.

  • Setup Eclipse Preferences
    • Window > Preferences... (or on Mac, Eclipse > Preferences...)
    • Select Maven
      • check on "Download Artifact Sources"
      • check on "Download Artifact JavaDoc"
  • Import latitude-json-oauth-sample project
    • File > Import...
    • Select "General > Existing Project into Workspace" and click "Next"
    • Click "Browse" next to "Select root directory", find [someDirectory]/google-api-java-client-samples/latitude-json-oauth-sample and click "Next"
    • Click "Finish"
  • Run
    • Right-click on project latitude-json-oauth-sample
    • Run As > Java Application
    • If asked, type "LatitudeSample" and click OK
